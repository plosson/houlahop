siteio

One command from folder to live site — on my server, not Vercel’s.

My agents kept producing sites with nowhere to put them. siteio deploys static sites, PocketBase backends, or full Docker apps to my own VPS — HTTPS, real URL, and rollback, all through a CLI an agent can drive.

agentio

One CLI to rule them all.

I didn’t want a different CLI for every communication channel — one for email, one for Slack, one for JIRA, each with its own auth and credential store. agentio is a single CLI for all of them: authenticate once, use it everywhere.

falco

Falco has no CLI and no MCP — so my agents couldn’t use it.

My invoicing runs on Falco, which only speaks web and mobile. For an agent, that means browser automation — which I won’t allow anywhere near my accounting.

falco is the missing interface: a CLI that syncs Peppol invoices, sales invoices, and payment status, read-only except for marking invoices paid, with --json output an agent can parse.
